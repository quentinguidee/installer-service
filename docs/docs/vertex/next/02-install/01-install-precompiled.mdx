# Install with binaries

<Tabs groupId="operating-systems">

<TabItem value="linux" label="Linux">

1. Install [Docker](https://docs.docker.com/desktop/install/linux-install/) or [Docker Engine](https://docs.docker.com/engine/install/).

2. Download the precompiled binaries for your system from [GitHub Releases](https://github.com/vertex-center/vertex/releases/latest).

    :::info
    - **Raspberry Pi** - Download the `linux_armv6` version.
    :::

3. Run the **Vertex Kernel as sudo**:
    ```bash
    sudo ./vertex-kernel
    ```

4. Run **Vertex** as a regular user:
    ```bash
    ./vertex
    ```

5. Open the UI in your browser: [http://localhost:7518](http://localhost:7518)

</TabItem>

<TabItem value="macos" label="macOS">

1. Install Docker:
    - With [Docker for macOS](https://docs.docker.com/desktop/install/mac-install/)
    - **Or** with Homebrew: `brew install --cask docker`.

2. Download the precompiled binaries for your system from [GitHub Releases](https://github.com/vertex-center/vertex/releases/latest).

    :::info
    - **Intel** - Download the `darwin_amd64` version.
    - **M1/M2** (or newer) - Download the `darwin_arm64` version.
    :::

3. Run the **Vertex Kernel as sudo**:
    ```bash
    sudo ./vertex-kernel
    ```

4. Run **Vertex** as a regular user:
    ```bash
    ./vertex
    ```

5. Open the UI in your browser: [http://localhost:7518](http://localhost:7518)

</TabItem>

<TabItem value="windows" label="Windows">

1. Install [Docker Desktop for Windows](https://docs.docker.com/desktop/install/windows-install/).

2. Download the precompiled binaries for your system from [GitHub Releases](https://github.com/vertex-center/vertex/releases/latest).

3. Run the **Vertex Kernel as administrator**:
    ```bash
    .\vertex-kernel.exe
    ```

4. Run **Vertex** as a regular user:
    ```bash
    .\vertex.exe
    ```

5. Open the UI in your browser: [http://localhost:7518](http://localhost:7518).

</TabItem>

</Tabs>

:::tip{About sudo}
To learn more about permissions, read [Security > Vertex Kernel](../security/vertex-kernel).
:::

## Flags and environment variables

The following flags and environment variables are available.

Note that the availability of some flags can depend on the target application.
An app that doesn't use a database, for example, won't have any database flags.

For a more detailed list of flags, use the `--help` flag.

| Flag                   | Environment variable | Description                                            |
|------------------------|----------------------|--------------------------------------------------------|
| `--version`            |                      | Print version and exit.                                |
| `--commit`             |                      | Print commit hash and exit.                            |
| `--date`               |                      | Print build date and exit.                             |
| `--host <host>`        | `VERTEX_HOST`        | Host to bind to.                                       |
| `--port <port>`        | `VERTEX_PORT`        | Port to bind to.                                       |
| `--db-driver <driver>` | `VERTEX_DB_DRIVER`   | Database driver to use. Can be `postgres` or `sqlite`. |
| `--db-host <host>`     | `VERTEX_DB_HOST`     | Database host to connect to.                           |
| `--db-port <port>`     | `VERTEX_DB_PORT`     | Database port to connect to.                           |
| `--db-user <user>`     | `VERTEX_DB_USER`     | Database user to connect with.                         |
| `--db-pass <pass>`     | `VERTEX_DB_PASS`     | Database password to connect with.                     |
| `--*-addr`             | `VERTEX_*_ADDR`      | Address of a service. Replace `*` with the service ID. |
